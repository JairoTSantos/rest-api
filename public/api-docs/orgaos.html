GET /api/orgaos

Esta rota retorna uma lista paginada de órgãos.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Parâmetros de consulta:

  • itens: O número de itens por página.
  • pagina: O número da página a ser retornada.
  • ordenarPor: O campo pelo qual os resultados serão ordenados.
  • ordem: A direção da ordenação (ASC ou DESC).

Exemplo de resposta:

        
          {
            "status": 200,
            "message": "1 órgãos encontrados",
            "data": [
              {
                "orgao_id": 1,
                "orgao_nome": "Nome do Órgão",
                "orgao_email": "orgao@example.com",
                // outros campos do órgão
              }
            ],
            "links": {
              "first": "/api/orgaos?itens=10&pagina=1&ordenarPor=orgao_nome&ordem=ASC",
              "self": "/api/orgaos?itens=10&pagina=2&ordenarPor=orgao_nome&ordem=ASC",
              "last": "/api/orgaos?itens=10&pagina=3&ordenarPor=orgao_nome&ordem=ASC"
            }
          }

GET /api/orgaos/{id}

Esta rota retorna um órgão específico pelo seu ID.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Parâmetros de consulta:

  • id: O ID do órgão.

Exemplo de resposta para um órgão encontrado:

        
          {
            "status": 200,
            "message": "Órgão encontrado",
            "data": {
              "orgao_id": 1,
              "orgao_nome": "Nome do Órgão",
              "orgao_email": "orgao@example.com",
              // outros campos do órgão
            }
          }

Exemplo de resposta para um órgão não encontrado:

        
          {
            "status": 404,
            "message": "Órgão não encontrado"
          }

POST /api/orgaos

Esta rota adiciona um novo órgão.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Corpo da solicitação:

        
          {
              "orgao_nome": "Nome do Órgão",
              "orgao_email": "orgao@example.com",
              // outros campos do órgão
          }

Exemplo de resposta para um órgão adicionado com sucesso:

        
          {
            "status": 200,
            "message": "Órgão adicionado com sucesso",
            "data": {
              "orgao_id": 123 // ID do órgão adicionado
            }
          }

Exemplo de resposta para um órgão já existente:

        
          {
            "status": 409,
            "message": "Órgão já inserido"
          }

Exemplo de resposta para um erro interno do servidor:

        
          {
            "status": 500,
            "message": "Erro interno do servidor",
            "error": "MENSAGEM_DE_ERRO"
          }
