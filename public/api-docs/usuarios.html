GET /api/usuarios

Esta rota retorna uma lista paginada de usuários.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Parâmetros de consulta:

  • itens: O número de itens por página.
  • pagina: O número da página a ser retornada.

Exemplo de resposta:

        
          {
            "status": 200,
            "message": "1 usuários encontrados",
            "data": [
              {
                "usuario_id": "453d9f00-b421-4b4a-b995-2bee9a61f0e4",
                "usuario_nome": "João Silva",
                "usuario_email": "joao.silva@example.com",
                "usuario_aniversario": "1990-05-15",
                "usuario_telefone": "123456789",
                "usuario_ativo": true,
                "usuario_nivel": 1,
                "usuario_criado_em": "2024-03-19T17:52:26.000Z",
                "usuario_atualizado_em": "2024-03-19T17:52:26.000Z"
              }
            ],
            "links": {
              "first": "/api/usuarios?itens=10&pagina=1",
              "self": "/api/usuarios?itens=10&pagina=2",
              "last": "/api/usuarios?itens=10&pagina=3"
            }
          }

GET /api/usuarios/{id}

Esta rota retorna um usuário específico pelo seu ID.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Parâmetros de consulta:

  • id: O ID do usuário.

Exemplo de resposta para um usuário encontrado:

        
          {
            "status": 200,
            "message": "Usuário encontrado",
            "data": [
            {
              "usuario_id": "453d9f00-b421-4b4a-b995-2bee9a61f0e4",
              "usuario_nome": "João Silva",
              "usuario_email": "joao.silva@example.com",
              "usuario_aniversario": "1990-05-15",
              "usuario_telefone": "123456789",
              "usuario_ativo": true,
              "usuario_nivel": 1,
              "usuario_criado_em": "2024-03-19T17:52:26.000Z",
              "usuario_atualizado_em": "2024-03-19T17:52:26.000Z"
            }
          ],
          }

Exemplo de resposta para um usuário não encontrado:

        
          {
            "status": 404,
            "message": "Usuário não encontrado"
          }

DELETE /api/usuarios/{id}

Esta rota exclui um usuário específico pelo seu ID.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Parâmetros de consulta:

  • id: O ID do usuário a ser excluído.

Exemplo de resposta para um usuário excluído com sucesso:

        
          {
            "status": 200,
            "message": "Usuário excluído com sucesso"
          }

Exemplo de resposta para um usuário não encontrado:

        
          {
            "status": 404,
            "message": "Usuário não encontrado"
          }

Exemplo de resposta para um erro interno do servidor:

        
          {
            "status": 500,
            "message": "Erro interno do servidor",
            "error": "MENSAGEM_DE_ERRO"
          }

POST /api/usuarios

Esta rota adiciona um novo usuário.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Corpo da solicitação:

          
            {
                "usuario_nomde": "string",
                "usuario_email": "usdsder@exaesddmplde.com",
                "usuario_senha": "string",
                "usuario_telefone": "09461549318",
                "usuario_aniversario": "2024-03-19",
                "usuario_nivel": "1",
                "usuario_ativo": true
              }

Exemplo de resposta para um usuário adicionado com sucesso:

          
            {
              "status": 200,
              "message": "Usuário adicionado com sucesso",
              "data": {
                "usuario_id": 123 // ID do usuário adicionado
              }
            }

Exemplo de resposta para um usuário já existente:

          
            {
              "status": 409,
              "message": "Usuário já inserido"
            }

Exemplo de resposta para um erro interno do servidor:

          
            {
              "status": 500,
              "message": "Erro interno do servidor",
              "error": "MENSAGEM_DE_ERRO"
            }

PUT /api/usuarios/{id}

Esta rota atualiza os dados de um usuário existente.

Requer autenticação: Sim (Token JWT no cabeçalho Authorization)

Para obter o token JWT, consulte /api/login.

Parâmetros de consulta:

  • id: O ID do usuário a ser atualizado.

Corpo da solicitação:

          
            {
              "usuario_nome": "Novo Nome do Usuário",
              "usuario_email": "novo_email@example.com",
              // Outros campos atualizados do usuário
            }

Exemplo de resposta para um usuário atualizado com sucesso:

          
            {
              "status": 200,
              "message": "Usuário atualizado com sucesso"
            }

Exemplo de resposta para um usuário não encontrado:

          
            {
              "status": 404,
              "message": "Usuário não encontrado"
            }

Exemplo de resposta para um erro interno do servidor:

          
            {
              "status": 500,
              "message": "Erro interno do servidor",
              "error": "MENSAGEM_DE_ERRO"
            }

Exemplo de resposta para tabelas atualizadas com sucesso:

            }

Exemplo de resposta para um erro interno do servidor:

          
            {
              "status": 500,
              "message": "Erro interno do servidor",
              "error": "MENSAGEM_DE_ERRO"
            }
