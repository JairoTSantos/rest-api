Voltar

POST /api/login

Esta rota permite que os usuários façam login no sistema.

Para fazer login, você deve enviar uma solicitação POST com um corpo contendo as seguintes informações:

  • email: O endereço de e-mail do usuário.
  • senha: A senha do usuário.

Exemplo de corpo da solicitação:

        
          {
            "email": "exemplo@email.com",
            "senha": "senha123"
          }

A rota retorna um objeto JSON com as seguintes informações:

  • status: O código de status da resposta.
  • message: Uma mensagem descrevendo o resultado da solicitação.
  • token: O token de acesso JWT, se o login for bem-sucedido.

Exemplo de resposta para um login bem-sucedido:

        
          {
            "status": 200,
            "message": "Login feito com sucesso",
            "token": "TOKEN_JWT"
          }

Exemplo de resposta para um login mal-sucedido (email incorreto):

        
          {
            "status": 404,
            "message": "Email incorreto"
          }

Exemplo de resposta para um login mal-sucedido (senha incorreta):

        
          {
            "status": 401,
            "message": "Senha incorreta"
          }

Exemplo de resposta para um erro interno do servidor:

        
          {
            "status": 500,
            "message": "Erro interno do servidor",
            "error": "MENSAGEM_DE_ERRO"
          }
